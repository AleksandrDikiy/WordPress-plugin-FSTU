## 🚀 Функціональні можливості (Модулі)

Плагін містить понад 70 модулів та форм, які розділені на три великі логічні блоки:

### 1. Реєстри ФСТУ
Ключовий модуль для автоматизації обліку членів та активів федерації.
* 👤 **Реєстр членів ФСТУ** (з детальною інформацією: досвід, розряди, членські квитки)
* ⚖️ **Реєстр суддів ФСТУ**
* 🏢 **Склад керівних органів ФСТУ**
* ⛵ **Вітрильні реєстри:** Реєстр капітанів, Реєстр стернових, Реєстр суден
* 🏅 **Реєстри відзнак**
* 🎓 **Семінари ФСТУ**

### 2. Структура ФСТУ
Управління ієрархією та органами самоврядування:
* Осередки федерації спортивного туризму
* Виконком та Президія
* Комісії з видів туризму
* Колегія суддів змагань з ВСТ
* Реєстр членів МКК (Маршрутно-кваліфікаційних комісій)
* Рада ветеранів та Ревізійна комісія
* З’їзд ФСТУ

### 3. Довідники (System Dictionaries)
Блок системних налаштувань для гнучкого управління даними (понад 20 довідників), серед яких:
* Види змагань, походів, складності та видів туризму
* Клуби, осередки, комісії та керівні органи
* Географія (Країни, області, населені пункти)
* Спортивні розряди та суддівські категорії
* Статуси карток/квитків, типи вітрильних залікових груп
* Реєстратори, Статистика та Користувачі онлайн

## 🏗 Архітектура та Стандарти Коду

Плагін переписано з урахуванням MVC-подібного підходу та суворих правил безпеки рівня WordPress VIP:

* **Модульність:** Кожен логічний блок має власні Controllers (`includes/`), Views (`views/`) та Assets (`js/`, `css/`).
* **Безпека (Security First):**
  * Перевірка Nonce (`check_ajax_referer`) для всіх AJAX-запитів.
  * Обов'язкова перевірка прав (`current_user_can`) перед виконанням дій.
  * Пізнє екранування (`esc_html`, `esc_attr`) перед виводом даних у шаблон.
* **База даних:** Абсолютно всі запити до БД виконуються ТІЛЬКИ через Prepared Statements (`$wpdb->prepare()`). Використовується `$wpdb->esc_like()` для безпечного пошуку.
* **JS/CSS:** Жодних inline-скриптів. Використовується делегування подій у jQuery та локалізація скриптів (`wp_localize_script`) для передачі бекенд-змінних.

## 📦 Встановлення, Деплой та Оновлення

Плагін використовує бібліотеку **[Plugin Update Checker](https://github.com/YahnisElsts/plugin-update-checker)**. Це означає, що вам **не потрібно** оновлювати його вручну через FTP чи завантаження ZIP-архівів.

### Початкове встановлення:
1. Завантажте актуальний реліз з GitHub.
2. Розпакуйте у директорію `wp-content/plugins/fstu`.
3. Активуйте плагін "FSTU" через адмін-панель WordPress.

### Процес оновлення (Деплой):
1. Всі зміни розробляються та тестуються у гілці `main`.
2. При випуску нової версії (створенні Release / Tag на GitHub), плагін автоматично надсилає сигнал системі WordPress.
3. Адміністратор сайту бачить стандартне повідомлення про оновлення в адмінці (`Консоль -> Оновлення`) та оновлює плагін в один клік, як і будь-який інший плагін з репозиторію WP.

## 🗄 Структура бази даних

Плагін використовує розширену (custom) структуру бази даних, яка працює паралельно зі стандартними таблицями WordPress. Таблиці зберігають історичні дані федерації (`S_Region`, `S_TourismType`, `UserClub`, `ApplicationShipTicket`, `Dues`, `DuesSail` тощо).
Усі міграції та зміни схеми керуються через вбудований у плагін механізм `dbDelta()`.
